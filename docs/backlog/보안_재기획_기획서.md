## 2. 채택/포기 결정 종합

### ✅ 이번 트랙 채택

| 분류 | 후보 | 위치 |
|---|---|---|
| A 기억형 | 마스터 비밀번호 (현재 유지) | wrappedDEK_master |
| A 기억형 | PIN 4/6자리 — 사용자 선택 (soft lock 재개봉용) | pinHash bcrypt |
| B 소유형 | 24단어 BIP39 (현재 유지 + UX 4종 개선) | wrappedDEK_recovery |
| B 소유형 | 이메일 매직 링크 (현재 유지, Phase 4 모달 그대로) | wrappedDEK_email |
| B 소유형 | TOTP — 비상 키 추가 (선택) | wrappedDEK_totp + wrappedTotpSecret |
| C 생체형 | 지문/Face ID — WebAuthn 디바이스별 등록 | biometricCredentials 컬렉션 |

### ⏸️ 보류 (베타 후)

- YubiKey 하드웨어 키
- Shamir 위임 (D 신뢰형) — DEK wrap 모델 전면 재설계 필요
- 패스(PASS)앱 — 사업자 등록·통신사 API 장벽
- SMS 핸드폰 인증 — 이메일 매직 링크와 중복·비용 부담
- 패턴 잠금 — 어깨너머 취약

### ❌ 제외

- 가상 키보드 — 다음 세션
- 공공장소 자동 감지 — 오감지 위험, 다음 세션

## 3. 데이터 모델 변경

### 3.1 신규 컬렉션 3종 (encryptionPolicy.js 확장)

```javascript
// 신규 1: 보안 설정 (모든 토글·시간·디폴트)
// users/{uid}/settings/security 단일 문서
securitySettings: {
    plaintext: [
        'id', 'updatedAt',

        // 자동잠금 모드
        'autoLockMode',          // 'single' | 'two-step'
        'softLockMinutes',       // 디폴트 5 (two-step일 때만 의미)
        'hardLockMinutes',       // 디폴트 30 (단일 모드는 이 값만)
        'backgroundLockMinutes', // 디폴트 3 (백그라운드 진입 시)

        // 인증 수단 토글 (사용자가 켜고 끔)
        'recovery24WordsEnabled',  // 디폴트 true
        'emailRecoveryEnabled',    // 디폴트 true (이미 존재)
        'totpEnabled',             // 디폴트 false
        'biometricEnabled',        // 디폴트 false (디바이스별 추가 시 자동 true)
        'pinEnabled',              // 디폴트 false
        'pinLength',               // 4 또는 6

        // 가리기 모드
        'sensitiveDataHidden',     // 현재 토글 상태
        'sensitiveMaskCategories'  // ['meditation', 'personNames', 'amounts', 'dotReasons']
    ],
    encrypted: [
        'pinHash',           // bcrypt 해시
        'wrappedTotpSecret'  // TOTP 시드를 마스터 키로 wrap
    ]
},

// 신규 2: 디바이스별 생체인증 자격 (1 사용자 N 디바이스)
biometricCredentials: {
    plaintext: [
        'id', 'userId',
        'credentialId',     // WebAuthn credentialId (디바이스 식별)
        'deviceName',       // "내 iPhone 15" — 사용자 입력
        'transports',       // ['internal', 'hybrid'] — WebAuthn 표준
        'registeredAt', 'lastUsedAt'
    ],
    encrypted: [
        'publicKey',                // WebAuthn 공개 키
        'wrappedDEK_biometric'      // DEK를 공개 키로 wrap
    ]
}

// 신규 3: TOTP — securitySettings 안에 흡수 (별도 컬렉션 X)
//   wrappedTotpSecret: TOTP 시드를 마스터 키로 wrap (마스터로만 풀림)
//   클라이언트는 마스터 풀고 들어와서 시드 꺼내 6자리 매칭
//   wrappedDEK_totp는 users/{uid} 문서에 추가
```

### 3.2 users/{uid} 문서 확장 (wrappedDEK 종류 추가)

기존:
```
wrappedDEK_master   (마스터 비밀번호로 wrap)
wrappedDEK_recovery (24단어로 wrap)
wrappedDEK_email    (이메일 슬롯으로 wrap)
```

추가:
```
wrappedDEK_totp       (TOTP 시드 기반 키로 wrap, 옵션)
```

지문/Face ID용 wrappedDEK_biometric은 디바이스별이라 `biometricCredentials` 컬렉션 안에.

### 3.3 데이터 형태·크기 변화

| 위치 | 변화 |
|---|---|
| users/{uid} | +200~400 바이트 (wrappedDEK_totp 1개) |
| securitySettings 신규 | ~500 바이트 (모든 설정 + pinHash) |
| biometricCredentials 디바이스당 | ~300 바이트 (공개 키 + wrappedDEK_biometric) |

→ 클라우드 저장 용량 거의 안 늘어남.

## 8. 5살 비유 데이터 흐름 다이어그램

### 8.1 현재 (마스터 비밀번호만)

```
[너의 머리] → 비밀번호 → [브라우저] Argon2id → 마스터 키
                                     │
                                     ▼ wrappedDEK_master 풀기
                                  DEK 🔑
                                     │
                                     ▼ 묵상 풀기
                                  화면 표시
```

### 8.2 신규 — 모든 인증 수단 결합

```
[잠금 해제 시도]
   │
   ├─ 지문/Face ID 등록됨? 우선 표시 ─┐
   ├─ TOTP 등록됨? 6자리 입력 ────────┤
   ├─ PIN 설정됨? 4/6자리 입력 ───────┤
   └─ 마스터 비밀번호 ─────────────────┤
                                       │
                                       ▼
                              [어느 경로로든 DEK 복원]
                                       │
                                       ▼
                                  화면 표시
```

### 8.3 가입 시 자동 등록 흐름

```
[가입]
   │
   ▼ 마스터 비밀번호 정함
   ▼ Argon2id로 마스터 키
   ▼ crypto.getRandomValues로 DEK 생성 (32바이트 무작위)
   │
   ├─ DEK를 마스터 키로 wrap → wrappedDEK_master ✅
   ├─ 24단어 BIP39 생성 → DEK를 24단어 기반 키로 wrap → wrappedDEK_recovery ✅
   ├─ 이메일 자동 등록 (Phase 3) → wrappedDEK_email ✅
   │
   ▼ 24단어 표시 화면
      ┌─ 위치별 슬롯 시각화
      ├─ [📄 인쇄 PDF]
      ├─ [📷 QR 코드 저장]
      └─ 체크리스트
   │
   ▼ 첫 화면 진입
```

### 8.4 두 단계 자동잠금 상태머신

```
UNLOCKED ─── 5분 무활동 ──→ SOFT_LOCK (DEK 메모리에 있음, UI만 가림)
   │                            │
   │                            ├─ PIN으로 1초 풀기 ──→ UNLOCKED
   │                            │
   │                            └─ 30분 무활동 또는 백그라운드 3분 ──→ HARD_LOCK
   │                                                                       │
   └────────────────── 30분 무활동 / 백그라운드 ─────→ HARD_LOCK            │
                                                          │                │
                                                          │                │
                                          DEK 메모리 폐기 ▼                │
                                                          │                │
                                          마스터 비밀번호 또는 ────────────┘
                                          24단어 / 이메일 / TOTP
                                                          │
                                                          ▼
                                                       UNLOCKED
```

## 9. 시나리오별 동작 (검증 체크리스트)

| 시나리오 | 기대 동작 |
|---|---|
| 신규 가입 | 24단어 표시 + 인쇄/QR + 체크리스트 + 이메일 자동 등록 |
| 일상 로그인 (지문 등록 후) | 지문 1회로 풀기 |
| 일상 로그인 (지문 없는 데스크탑) | 비밀번호 또는 TOTP 또는 PIN |
| 5분 자리 비움 | soft lock → PIN으로 빠르게 |
| 30분 자리 비움 | hard lock → 비밀번호 또는 비상 키 |
| 백그라운드 3분 | hard lock 즉시 |
| 비밀번호 까먹음 | 3종 비상 키 분기 화면 → 5살 카피로 차이 설명 |

| TOTP 핸드폰 잃음 | 24단어 또는 이메일로 풀고 TOTP 재등록 |
