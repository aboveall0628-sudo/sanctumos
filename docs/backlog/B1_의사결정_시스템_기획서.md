## 1. 개요

> **원칙 = 법조항, 의사결정 사례 = 판례** (사용자 수업 메모 원문)

사용자가 선택의 기로에 섰을 때, 저장된 원칙·판례를 AI가 참조해 사용자 자신의 분별을 돕는 시스템.

### 핵심 설계 철학
**"평가보다 인과"** — 주관적 평가는 시점에 따라 뒤집힌다.
- 판례에 "잘했다/후회" 같은 평가 필드 두지 않음
- 대신 **인과의 가지** + **시점 스냅샷** + **수정 로그** 우선 추적
- 법체계 비유: 법 개정 시 "왜 바뀌었나"의 입법이유서가 진단 자료

---

## 2. 범위와 작성 주체

| 항목 | 결정 |
|---|---|
| 다루는 결정 범주 | 일상 / 관계 / 신앙 / 인생 — **전 범주 커버** |
| 원칙·판례 작성 주체 | **합작** — 사용자 초안 + AI 수정/제안 결합 |
| 작동 시점 | **사전(보조) + 사후(기록) 둘 다** (UI 분리) |
| 데이터 암호화 | 자동 E2EE 적용 (Sanctum OS 기본 정책) |

## 3. 데이터 모델

### 원칙(Principle)
```
- id
- title              "약속한 일은 작더라도 지킨다"
- body               본문
- category           일상 / 관계 / 신앙 / 인생 / meta
- strength           절대 / 강한 / 약한
- conditions         적용 조건
- source             가치관 / 성경 / 책 / AI 제안
- createdBy          user / ai_drafted_user_confirmed
- createdAt
- linkedDots         이 원칙으로 만들어진 도트 (역참조)
- linkedPrecedents   이 원칙이 적용된 판례
- revisionLog: [
    {
      revisedAt,
      previousBody / previousStrength,
      reason,
      triggeredBy: {
        type: "precedent" | "external_input" | "reflection" | "advice"
        reference
      }
    }
  ]
```

### 판례(Precedent)
```
- id
- situation              당시 상황
- decision               결정 내용
- decidedAt
- principlesAtTime: [    그 시점의 원칙 스냅샷 (강도/본문 포함)
    { principleId, strengthAtTime, bodyAtTime }
  ]
- contextNote            자유 메모 (평가 아닌 그 당시의 결)
- linkedDots             이 결정 이후 발생한 도트 (인과 가지)
- linkedPrecedents       이 결정에서 파생된 후속 결정
- linkedWorkflow         이 결정에서 생성된 워크플로우/목표
```

### 메타 원칙
별도 모델 아님 — 원칙 `category` 에 **"meta"** 추가. 일반 원칙과 동일한 구조이되 적용 대상이 "다른 원칙들의 우선순위 판단"인 점이 다름.

## 4. AI 활용 흐름

### 응답 형태: 소크라테스법
AI는 답이 아닌 **질문**을 던져 사용자의 내적 분별을 돕는다.

### 운영 가이드라인
```
[사전 모드]
1. 질문은 유한 — 최대 5개
2. 모든 질문은 관련 원칙/판례를 자연스럽게 인용
   예: "약속한 일은 지킨다는 원칙이 있으셨죠.
        지난 3월의 비슷한 결정에서는 어떻게 적용하셨나요?"
3. 5번째 질문 후 결론 미정 → 자동 "정리해드릴까요?" 제안
4. 항상 클로징 질문: "그래서 어떻게 결정하시겠어요?"

[사후 모드]
1. 소크라테스 X — 그냥 기록 폼
2. 결정이 기존 원칙과 충돌하면 한 줄 알림
3. 자동 판례화 시 사용자 확인 필수
```

### AI 의견 제시 모드 전환 트리거
- ① 사용자가 명시적으로 요청 ("의견 줘", "어떻게 생각해")
- ② 5번째 질문 후에도 결론 미정
- ③ 원칙끼리 명백히 충돌해서 사용자가 괴로워할 때

## 5. 메타 원칙 (B-3) 처리

### 부트스트랩 — 원칙이 0개일 때
**성경 구절/가치관 라이브러리에서 후보 추출** (신앙 특화).
- 사용자 상황을 보고 관련 성경 구절 또는 일반 가치 키워드에서 후보 원칙 제시
- 사용자가 선택/수정 → 원칙 등록
- "남의 원칙" 느낌을 줄이려면 후보 제시 시 출처를 명확히 (성경 X장 Y절, ○○책 인용 등)

### 충돌 처리 — 4단계 폴백
```
1. strength 우선         절대 > 강한 > 약한
                        동급이면 ↓
2. 메타 원칙 적용         "절대 원칙 충돌 시 신앙 우선" 같은 명시 규칙
                        없거나 모호하면 ↓
3. 소크라테스 모드        AI가 5개 질문으로 사용자 자체 우선순위 발견 도움
                        결과 ↓
4. 자동 판례화            어떻게 풀었든 판례로 기록 → 다음 충돌 시 참조 +
                        반복되면 메타 원칙 후보로 제안
```

**무한 회귀 방지**: 메타 원칙끼리 충돌하면 메타-메타로 가지 않고 **무조건 3단계(소크라테스) 직행**.

## 7. 다른 모듈에 미치는 영향

이 시스템은 Sanctum OS의 **기반 모듈** 위치를 차지함. 다음 모듈들이 이 위에 얹힐 수 있음:

- **Reports 모듈** → "의사결정 진화 리포트" 추가 가능 (원칙 수정 경로 분석)
- **인물 카드** → 관계 결정 판례와 직접 연동
- **도트 시스템** → 각 도트가 어떤 원칙·판례에서 비롯됐는지 역추적 가능
- **목표/워크플로우** → 판례에서 자동 워크플로우 생성 (수업 메모 원문)
- **DigiSoul** → 친밀도 마이너스 + deceptionCount 컨셉이 동일, B-5와 통합 설계 검토 가치
