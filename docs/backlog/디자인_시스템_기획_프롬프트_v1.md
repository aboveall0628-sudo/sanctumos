# 사용자 톤 가이드 (모든 UI 카피) — 🚨 TDS 무조건 준수 의무

> **모든 UI 카피·라벨·버튼·toast·에러 메시지·alt 텍스트·tooltip 은 무조건 TDS(Toss Design System) UX 라이팅 원칙을 따라야 해요. 산출물 검증 단계마다 "TDS 정합도 자체 검사"를 필수로 진행하고, 카피 한 줄 한 줄 어디가 TDS 원칙의 어느 자리와 어울리는지 명시해주세요. 다른 톤(딱딱한 격식체·시스템 메시지 톤·과장된 마케팅 톤)은 절대 허용 안 됨.**

## TDS 핵심 라이팅 원칙 (무조건 적용)

1. **사용자 입장에서 쓰기** — 시스템 동작 X, 사용자 결과·감정 O
   - ❌ "비밀번호 변경 처리됨" → ✅ "비밀번호 새로 정했어요"
2. **간결하게** — 한 문장 길이 짧게 (8~16자 권장). 부사·접속사 최소
   - ❌ "여기서 새로운 비밀번호를 입력하실 수 있습니다" → ✅ "새 비밀번호 적어주세요"
3. **구체적으로** — 추상어·전문 용어·외래어 우회. 사용자가 머릿속에 즉시 그릴 수 있는 단어
   - ❌ "오류 발생" → ✅ "잠시 멈췄어요"
   - ❌ "프로세스 진행 중" → ✅ "정리하고 있어요"
4. **친근하게** — 해요체·캐주얼 경어·친구 톤. 격식체·존대 끝 ~하십시오·~합니다 금지
5. **솔직하게** — 모호한 표현 X·과장 X·"최고의" 류 마케팅 단어 금지
   - ❌ "최고의 묵상 경험" → ✅ "오늘의 묵상"
6. **거리를 두고 평가 X** — 사용자를 평가하지 않음. 행동만 거울처럼 비추기
   - ❌ "잘하고 있어요" → ✅ "오늘도 함께해주셨네요"

## 어체 강제 사항

- **해요체** (~해요, ~예요) 무조건 — 명령형(~하세요)·의문형(~인가요)·격식체(~합니다·~하십시오) 절대 금지
- **능동·긍정** — 부정 표현 우회 ("저장 실패" X → "저장에 문제가 있었어요" O)
- **캐주얼 경어** — 친구처럼 가까운 톤. 거리감 두는 격식 X
- **금지 어휘**: "박-" 어간 6종 (박힌·박혀·박힘·박았·박을·박기) — 모든 카피·코드 주석·문서에 절대 금지. 대체어 카탈로그: 자리잡힌·자리잡혀·자리잡힘·넣었·정할·정하기·매듭짓고
- **평가 라벨 금지** — "능력 N점" 류 평가형 카피는 "함께 느낀 만족도 N%" 결로 우회
- **마케팅 과장 금지** — "최고", "혁신적", "AI 기반의" 류 강조 단어 절대 금지. 사용자가 받는 실제 가치만 담백하게

## TDS 정합 검사 예시 (산출물에서 매 카피마다 적용)

| ❌ 안 됨 | ✅ TDS 정합 | 적용된 원칙 |
|---|---|---|
| 비밀번호를 입력하세요 | 비밀번호 적어주세요 | 친근 + 해요체 |
| 저장에 실패했습니다 | 저장에 문제가 있었어요. 잠시 후 다시 시도해주세요 | 사용자 입장 + 솔직 |
| 이 기능은 잠겨 있습니다 | 아직 열리지 않은 자리예요 | 사용자 입장 + 친근 |
| 새 묵상을 작성하시겠습니까? | 오늘 묵상 시작할까요? | 구체 + 간결 + 친근 |
| 데이터 처리 중입니다 | 정리하고 있어요 | 구체 + 친근 |
| 알림 설정이 완료되었습니다 | 알림 자리잡았어요 | 사용자 입장 + 간결 + 친근 |
| 회원가입을 환영합니다! | 함께해주셔서 감사해요 | 솔직 + 평가 X + 친근 |

## 산출물별 TDS 검사 필수 자리

- **산출물 1 토큰**: 카피 거의 없음. 통과
- **산출물 2 컴포넌트**: 모든 버튼 라벨·placeholder·에러 메시지·툴팁 — **카피별 TDS 원칙 매핑 표** 필수
- **산출물 3 페이지 mockup**: 헤더·subtitle·CTA·상태 메시지·빈 상태 카피 — **페이지마다 TDS 검사표** 필수
- **산출물 4 인터랙션**: toast·alert·confirm·발화 카피 — **카피 카탈로그 + 원칙 매핑** 필수
- **산출물 5 접근성**: aria-label·alt 텍스트도 TDS 정합 필수

**각 산출물 끝에 "TDS 정합 자체 검사 결과" 한 섹션을 의무로 포함하고, 발견된 위반·우회 결정을 명시해주세요.**

# 현재 디자인 톤 (Stitch 결과 — 살림)

- **모노톤 기조**: 무채색 grayscale 중심. 형광색·무지개 금지
- **Soft accent**: brand-accent 1색 (현재: blue-gray 계열) — 강조 자리에만 절제 사용
- **타이포**: Noto Serif KR (헤더·AI 발화·묵상 본문) + Pretendard (라벨·버튼·본문). serif와 sans-serif 의도적 분리
- **여백**: 절제된 여백 (1.5rem 기준 단위). 카드 내부 패딩 일관성
- **라운드**: 0.5~0.85rem 범위. 모달 0.85rem, 카드 0.65rem, 버튼 0.5rem
- **그림자**: 거의 사용 X. border 1px 자연 톤으로 대체

# 톤 변주 탐색 가이드 (N=2 결정)

현재 톤은 살리되 다음 3 자리에서 변주 후보 1~3개씩 던져주세요:

1. **Accent 색 변주** — 현재 blue-gray 1색 → 신앙 톤에 어울리는 후보 2~3개 (예: warm beige·muted olive·dusty lavender). 라이트·다크 모드 각각.
2. **라운드 변주** — 현재 0.5~0.85rem → 더 절제(0.25rem)·더 부드러움(1.2rem) 비교
3. **간격 변주** — 현재 1.5rem 단위 → 1.25rem 컴팩트·1.75rem 여유 비교

각 변주마다:
- 정체성 정합도 (걸어다니는 성경·동행 친구 톤과 어울리는가)
- 영적 톤 vs 생산성 도구 톤 균형
- 모바일 가독성
세 기준으로 평가 추천.

# 제약·견제 자리

1. **박-어간 6종 절대 금지** (박힌·박혀·박힘·박았·박을·박기). 대체어: 자리잡힌·자리잡혀·자리잡힘·넣었·정할·정하기
2. **평가 라벨 금지**: "능력 N점" → "함께 느낀 만족도 N%" 결로 우회
3. **게임 톤 vs 영적 톤 균형**:
   - 진입 자리(설정 카드·발견 자리) = 살짝 게임 톤 OK (✨ 별·발견의 재미)
   - 본문(카드 안 콘텐츠) = 영적 톤 (회고·간증·동행 친구)
4. **visibility 3단계** 표현: 🌍 공개 / 🤝 친한 사이 / 🔒 비공개 — 이모지 + 한국어
5. **정죄 톤 금지** (회복의 자리·약속 어김 자리): 무조건 부드러움
6. **무지개·형광색·강한 그라데이션 금지**: 모노톤 + soft accent 유지
