## 1. 배경

### 1.1 사용자 통증 (Q1 답)
- **시차 통증**: 저녁 묵상 시점에 그날 거래가 기억 안 남
- **도트 매칭 통증**: 도트 평가 시 💰 칩 → "이 시간에 결제 있었나" 매번 확인
- **은행 앱 왕복 통증**: 정확 금액 보러 다른 앱 갔다 옴
- **일정 외 지출 통증**: 도트와 매칭 안 되는 지출은 어디 적나

### 1.2 가이드 별 (사용자 명시)

> **"도트(경제) = 돈을 어디에 얼마나 쓰는지 데이터화 → 현상을 거울처럼 → 문제 정확히 파악 → 원인 명확하게 진단"**

평가 X. **거울 + 인과 + 원칙 도출**. 이 별 위에서 모든 결정 정렬.

### 1.3 핵심 통찰

**"거래 입력 시점이 묵상 시점과 겹치면 묵상이 영수증 정리 시간으로 변질된다."**

→ 트랙 1.a 의 진짜 과제: 거래 입력과 묵상을 정중하게 분리하되, 거울 정확성은 잃지 않기.

## 4. 트랙 1.a 골격

### 4.1 이벤트 도트 모델 신설

| 도트 종류 | 비유 | 용도 |
|---|---|---|
| 일정 도트 (현재) | 🎬 영화 — 시작·끝, 그 안의 활동 | 사무실 출근, 친구 만남 |
| **이벤트 도트** (신규) | 📸 사진 — 그 순간 찰칵 | 거래 · 말 한 마디 · 자동 결제 |

**필드 (초안)**:
```
{
  id, userId, timestamp, kind: 'event',
  payload: {
    type: 'transaction' | 'speech' | 'other',
    ...type별 페이로드
  },
  linkedScheduleDotId?: string,  // 일정 도트 안 입력 시 자동 매칭
  encryptedPayload  // 영적 안전장치
}
```

**원칙**:
- 일정 도트와 **독립** 작동 (결제·일정 미스매치 배려)
- payload 확장 가능 — 미래 다른 type 흡수 (말 한 마디 / 사진 / 기타)

### 4.2 거래 타입 9종

| 대분류 | 타입 | 비유 | 잔액 | 통계 |
|---|---|---|---|---|
| 수입 | 활성 (노동) | 일해서 번 돈 | + | 수입 합계 |
| 수입 | 비활성 (자산) | 내 돈이 일해서 번 돈 (배당·이자·임대) | + | 수입 합계 |
| 수입 | 매매 차익 (±) | 사고 팔아서 남은/잃은 돈 (손실 −) | ± | 수입 합계 (별도 표시) |
| 지출 | 일반 | 쓴 돈 | − | 지출 합계 |
| 지출 | 세금 ⛪ | "가이사의 것은 가이사에게" | − | 별도 칸 |
| 지출 | 헌금 ⛪ | "하나님의 것은 하나님께" | − | 별도 칸 (영적 톤) |
| 지출 | 고정비 | 구독·자동이체 | − | 지출 합계 (별도 표시) |
| 이체 | 내부 | 내 통장 사이 (A→B) | A− / B+ | **지출 합계 X** |
| 이체 | 외부 | 다른 사람에게 보낸 돈 (메모 필수) | − | **지출 합계 자동 합산** |

**원칙**:
- 이체는 잔액 이동 ≠ 지출 (분식회계 방지)
- 외부 이체는 자동으로 지출 합산 (사용자 분류 부담 X)
- 세금·헌금은 별도 통계 칸 (영적 톤 + 가시성)

### 4.3 매매 거래 = 의사결정 흔적 (B-1 연결)

사용자 명시:
> *"이거 왜 그랬는지 원인 파악 → 무슨 교훈을 얻었고 → 다음에 뭘 조심해야 하고 → 원칙 세울 수 있게"*

**매매 입력 모달 필드 3개 추가**:
- `tradeReason` — 왜 매수·매도했나 (text)
- `tradeLesson` — 무엇을 배웠나 (text)
- `linkedPrecedentId?` — 📜 "분별의 자리로" 옵션 버튼 (decisionGate 호출)

**원칙**:
- 자동 호출 X — 사용자가 누르면 분별의 자리 'goal-edit' 비슷한 모드 호출
- 같은 자산·같은 패턴 매매 그룹핑 → 후속 트랙
- linkedPrecedentId 양방향 역참조 (precedent.linkedTradeIds 같은 회로)

### 4.4 매칭 정책

| 거래 입력 시점 | 매칭 처리 |
|---|---|
| 일정 도트 안 (예: 도트 평가 모달에서 💰 칩) | **자동 매칭** (linkedScheduleDotId 자동 박힘) |
| 이벤트 도트 안 (예: 빠른 입력 D) | 독립 — 매칭 X |
| 일정 도트 외 진입점 (지금까지의 quickAdd) | **미매칭 박스**로 들어감 → 주간 회고에서 일괄 정리 |

**원칙**:
- 시스템에서 강제 매칭 X — 사용자 결정
- 미매칭은 "도트 없는 거래" 로 그대로 살아남음 (거울 정확성)

### 4.5 이체 UI — 하이브리드 자동완성

**받는 곳 필드 자동완성** (인물·조직 자동완성 컴포넌트 재활용):

```
받는 곳: [   타이핑    ]
        ┌─────────────────────────────┐
        │ 🏦 내부 이체                  │
        │   • 카카오뱅크 입출금          │
        │   • 토스 적금                  │
        │                              │
        │ 👤 외부 (전에 보낸 사람)        │
        │   • 김ㅇㅇ (3건 이력)          │
        │   • ㅇㅇ 가게                  │
        │                              │
        │ ➕ 신규로 적기...              │
        └─────────────────────────────┘
```

- 1순위: 본인 등록 통장 (자동 '내부 이체')
- 2순위: 과거 외부 수취인 이력 (자동 '외부 이체', placeholder 마지막 메모)
- 신규 외부: 메모 **필수** (왜 보냈는지)

**구현 노트**:
- score 매칭·초성·편집거리 — 인물·조직 자동완성 [v6](.) 패턴 재활용
- 외부 수취인은 자동 누적 → 미니 카드 모델 (별도 컬렉션 또는 personRoles 흡수 검토)
- UI 디테일은 코드 진입 후 직접 써보고 베타 직전 UX 일괄 점검에서 다듬기

### 4.6 다중 통화 — B 안 (베타 후)
- 시작 시 사용 통화 1회 설정 (currency 필드)
- 환율 변환·실시간 시세 → 베타 후 트랙 2
- 베타 유로 사용자도 시작 시 EUR 단일 설정으로 충분

### 4.7 영적 안전장치 (유지, 깨지 않음)
- AMOUNT_BUCKETS 디폴트 마스킹
- exactAmount 토글
- 카테고리 랭킹·부채 비율 X
- 헌금·세금 별도 톤
- 거액 거래 묵상 안내 (강제 X, 옵션)
