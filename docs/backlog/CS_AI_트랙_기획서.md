## §1. 트랙 본질

베타 사용자가 풍선(§8-1)으로 의견·불편·아이디어를 알려줄 때, **SWAN이라는 AI 에이전트가 채팅으로 맥락을 풀어내고**, 종료 시점에 자동 요약·분류 + 라벨 부착해서 Firestore에 저장합니다. Swan(사용자)이 인앱 관리자 페이지에서 블로그 리스트로 묶음을 보고, 복사해서 Claude한테 붙여넣으면 Claude가 분류·우선순위·작업 순서 정리 후 작업 사이클이 돕니다.

**중요 단순화 (2026-05-15)**: 거대한 인프라(Slack 자동 알림·Sheets 자동 추가·복잡한 상태 토글) 없이 *Claude가 어차피 분류·정리한다는 가정* 위에 시스템 가볍게 유지.

## §3. 전체 흐름

```
1. 베타 사용자가 풍선 (§8-1) 클릭
        ↓
   SWAN 채팅창 열림 — 먼저 말 거는 인사
        ↓
   다중 턴 대화 (사용자 ↔ SWAN, 최대 12턴)
   · 사용자가 모호하면 SWAN이 6단계 분해 질문
   · 사용자가 명확하면 보충 질문만
        ↓
   종료 조건
   · 5분 무응답 자동 종료
   · [보내기] 버튼 명시 종료
        ↓
   Gemini 자동 요약 + 분류
        ↓
2. Firestore feedbacks 컬렉션 저장
        ↓
3. Swan이 인앱 관리자 페이지에서 확인
   · 블로그 리스트 형식 (시간순)
   · ●○ 읽음 표시 · 검색·필터·정렬
   · 상세 클릭 → 대화 원본·요약·라벨
        ↓
4. 📋 복사 (단건 또는 묶음) → Claude
        ↓
   Claude가 분류·중복·우선순위·작업 순서 정리
        ↓
5. Swan이 정리본 보고 작업 순서 결정 → 작업
```

## §5. SWAN 시스템 프롬프트 1차

```
당신은 "SWAN"입니다 — Sanctum OS의 피드백 도우미입니다.

베타 사용자가 풍선을 통해 의견·불편·아이디어를 알려주려고 합니다.
당신의 역할: 사용자가 무엇이 불편하거나 어색했는지를 정확하고
명확하게 풀어내는 것입니다.

응대 톤:
- 평소 인사·진행: 해요체 친근하게
  ("안녕하세요. 오늘 어떤 부분을 알려주실 거예요?")
- 질문: 소크라테스식으로 사용자가 스스로 명확히 표현하도록
  ("그때 어떤 일을 하시려고 했어요?",
   "기대했던 동작과 실제 결과가 어떻게 달랐어요?")

질문 흐름 (모호한 한 줄이 오면 6단계 분해):
1. 어느 화면에서 그러셨어요?
2. 무엇을 하시려고 했어요?
3. 어떤 일이 일어났어요? (기대 vs 실제)
4. 다시 시도해 보셨어요? 재현되나요?
5. 처음 겪으시는 건가요, 반복되나요?
6. 더 알려주고 싶으신 게 있어요?

사용자가 처음부터 자세히 설명했으면 위 흐름 일부 건너뛰어도 됩니다.
대화는 최대 12턴 안에서 정리합니다.

금지:
- "박-" 어간 단어 절대 사용 안 함 (박힌·박혀·박힘·박았·박을·박기 등)
- 명령형 ("~하세요" 강한 어조) 안 씀
- 격식 존댓말 ("~하시겠습니까") 안 씀
```

## §6. 자동 요약·분류 호출

대화 종료 시점에 별도 LLM 호출 1회:

```
[프롬프트]
다음은 베타 사용자와 SWAN의 대화입니다.

{대화 원본}

다음 JSON 형식으로만 답하세요:
{
  "summary": "2~3줄 요약",
  "category": "error" | "ux_ui" | "feature_request" | "other",
  "confidence": 0.0~1.0
}
```

비용 가드: 호출 1회 추가 = 토큰 약 1~2K, Gemini Flash 기준 무시할 수준.

### §7.3 권한 게이트

```
[시각 차단 — 클라이언트]
if (currentUser.uid !== SWAN_ADMIN_UID) {
  return null;  // 관리자 페이지 자체 안 보임
}

[데이터 차단 — Firestore 보안 규칙]
match /users/{userId}/feedbacks/{feedbackId} {
  allow read: if request.auth.uid == SWAN_ADMIN_UID
              || request.auth.uid == userId;
  allow write: if request.auth.uid == userId
               && resource.data.userId == userId;
  allow update: if request.auth.uid == SWAN_ADMIN_UID;  // 상태 토글
}
```
