# Health Check 체크리스트
**용도**: 큰 트랙이 끝날 때마다 진행하는 "한숨 돌리기" 점검. 매번 같은 체크리스트로 진행해 일관성 보장.

> **타이밍**: 큰 트랙 끝 → Health Check (30분~1시간) → 다음 트랙 진입
> **첫 경험**: 워크플로우 트랙 STEP 2 (일일 의식 웹) 완료 후

---

## 📋 Health Check 3종 (혼합 옵션 A + 부분 B)

### 1️⃣ 기획 검토 (매번, 가볍게)
**왜**: 정체성이 가장 흔들리기 쉬움. 새 트랙마다 충돌 위험.

체크:
- [ ] **시스템 정체성 일치 확인** — 최근 트랙이 "걸어다니는 성경" / "거대한 유기체" / "평가보다 인과" / "묵상으로 돌아가기" 중 하나라도 흔든 점 없는가?
- [ ] **기획서 간 일관성** — 최근 작업이 기존 기획서와 충돌하지 않는가? (docs/backlog/ 안 기획서들 훑기)
- [ ] **새 패턴 발견** — 트랙 진행 중 등장한 반복 패턴이 있나? (예: AI 질의응답 부트스트랩 패턴이 본인 프로필/의사결정/베타 온보딩 셋에 공통)
- [ ] **메모리 정합성** — feedback / project 메모가 현재 작업과 모순되지 않는가?

처리:
- 새 통찰 → 메모리 feedback 추가 또는 기존 feedback 갱신
- 기획서 충돌 → 기획서 패치 또는 별도 백로그로 보관
- 정체성 표류 감지 → 즉시 사용자와 합의 (다음 트랙 진입 보류 가능)

---

### 2️⃣ 기능 점검 (매번, 가볍게)
**왜**: 회귀 누적 방지. 골든 패스 검증.

체크:
- [ ] **회귀 테스트** — 최근 트랙에서 만진 코드 외에, 기존 기능들이 그대로 작동하는가?
  · 도트 생성·기록 / 묵상 진입 / 알람 / 인물 카드 / 리포트 (각 계층)
- [ ] **골든 패스 1회 통과** — "도트 만들기 → 묵상 → 일일 의식 → 내일 도트 분배" 전체 흐름이 막힘 없는가?
- [ ] **알려진 버그 갱신** — `project_known_issues.md` 새 항목 추가 or 해결 표시
- [ ] **콘솔 에러** — 브라우저 콘솔에 새 에러 발생 안 했는가?

처리:
- 즉시 처리 가능한 작은 버그 → 그 자리에서 패치 + 푸시
- 큰 회귀 → 별도 트랙으로 분리, 다음 우선순위
- 알려진 이슈 → `project_known_issues.md` 갱신

---

### 3️⃣ UI 메모만 (손보기는 베타 직전에)
**왜**: 너무 일찍 손대면 기능 변경에 헛수고. 베타 직전 일괄 처리. 그 사이 발견된 사항만 메모.

체크:
- [ ] **일관성 깨진 곳 메모** — 폰트·여백·버튼 스타일이 다른 화면들
- [ ] **모바일/웹 어긋남 메모** — 같은 기능이 두 디바이스에서 다르게 보이는 곳
- [ ] **디자인 시스템 일탈 메모** — 임시로 박힌 스타일들
- [ ] **사용자 불편 메모** — 본인이 매일 쓰면서 느낀 작은 거슬림

처리:
- **즉시 손보지 않음** (예외: 명백한 사용 불가 버그)
- `docs/backlog/UI_점검_백로그.md`에 누적
- 베타 출시 직전 대점검 시 일괄 처리
